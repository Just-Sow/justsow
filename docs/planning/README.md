# Planning

This directory stores persistent high-level planning context for work that spans multiple sessions or agents.

## Purpose

Use this area for:

- roadmap direction
- prioritized backlog state
- active epic definitions
- cross-session scope, dependencies, and open questions

Do not use this area for:

- session-specific implementation plans
- step-by-step work logs
- routine status narration
- temporary notes better kept in agent context

## Structure

- `roadmap.md` for major phases and sequencing
- `backlog.md` for prioritized work items
- `active-epics/` for larger workstreams that need durable shared context
- `completed-epics/` for finished workstreams kept for reference
- `inactive-epics/` for workstreams that are paused or deferred

## Planning Rule

Planning files should stay concise and durable. They are shared context for future sessions, not a transcript of execution.
