# Account And Security

## Account Model

The platform should support:

- standard account creation
- sign-in
- password reset
- email verification
- account claiming for manually created sower records
- two-factor authentication

## Identity Rules

- one person may have multiple roles
- manually created sower records should be claimable by the real user later
- account claiming must preserve historical seed records

## Email Use Cases

- account creation
- email verification
- password setup for claimed accounts
- password reset
- account email update
- two-factor authentication flows
- project workflow notifications

## Security Expectations

- role-based access control
- private donor contact information
- audit trail for sensitive actions
- secure account recovery
- appropriate protection for approval and role-assignment actions
